Key rules for ALL changes:

1. `.pyodide(...)` with ONLY browser execution params (run_through_pyodide, environment_initialization_code, environment_initialization_code_filepath, on_game_step_code, packages_to_install, restart_pyodide) becomes `.runtime(...)` with identical params.

2. `.pyodide(...)` containing BOTH browser execution AND multiplayer/sync params must be SPLIT into `.runtime(...)` + `.multiplayer(...)`. Browser exec params go to .runtime(), everything else (multiplayer, server_authoritative, state_broadcast_interval, input_buffer_size, input_delay, input_confirmation_timeout_ms, realtime_mode) goes to .multiplayer().

3. `.user_experience(...)` with ONLY content params (scene_header, scene_body, scene_body_filepath, in_game_scene_body, in_game_scene_body_filepath, game_page_html_fn) becomes `.content(...)` with identical params.

4. `.user_experience(...)` containing BOTH content AND waitroom params must be SPLIT into `.content(...)` + `.waitroom(...)`. Content params go to .content(). Waitroom params go to .waitroom() WITH shortened names: waitroom_timeout -> timeout, waitroom_timeout_redirect_url -> timeout_redirect_url, waitroom_timeout_scene_id -> timeout_scene_id, waitroom_timeout_message -> timeout_message.

5. Update TEXT references too -- not just code blocks. Section headers, inline backtick references, and prose that mention old method names must be updated. For example, a section titled ".pyodide()" becomes ".runtime()", and text like "call `.pyodide()`" becomes "call `.runtime()`".

**quick_start.rst:** 3 references -- one .user_experience() -> .content(), one .pyodide() code block -> .runtime(), one text reference `.pyodide()` -> `.runtime()`.
